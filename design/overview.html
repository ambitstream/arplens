ArpLens

Recreate standard arpeggios from audio.

Get ready-to-use arpeggiator settings for your favorite synth, plugin or DAW. This file is the complete product & UI brief for design work — sourced verbatim from the frozen specification (v2.1). Treat quoted copy as literal, not a paraphrase target.

What ArpLens is / is not

ArpLens analyzes a short audio fragment containing a standard synthesizer arpeggio and reconstructs the arpeggiator settings required to recreate it — not a transcription tool, not a MIDI extraction tool. It reconstructs arpeggiator settings, not performances, and reconstructs how the arpeggiator was programmed, not the original synth sound.

Typical user questions it answers: which notes were held, which style, what rate, how many octaves, what BPM.

Terminology (do not blur these)

Uploaded Audio
The whole file the user uploads. Up to 200 MB / 10 minutes. No duration constraint at this step.
Focus Region
A user-selected ≤1 minute window, only required if the uploaded audio is longer than 1 minute.
Loop
The final 3–20 second fragment selected inside the Focus Region. This is the ONLY place a duration range applies.
Input Notes
The notes the musician held, fed into the arpeggiator (e.g. C, D#, G).
Generated Sequence
What the Style Registry produces from Input Notes + Style + Octaves.
Detected Sequence
What the app shows the user — either the generated sequence (complete result) or the quantized transcription (partial result).
Confidence
One overall band — High / Medium / Low — never a numeric score in the UI.

Full workflow (7 steps)

  1. Upload — drag & drop or click to browse. MP3 / WAV / M4A. No duration limit shown here beyond the max-file warning.
  2. Focus Region — only if audio > 1 minute; user selects max 1 minute. Skipped entirely for shorter files.
  3. Loop Selection — inside the Focus Region; 3–20 seconds; should loop seamlessly; selection quality drives analysis quality.
  4. Analyze — one primary button.
  5. Review — results appear only after analysis completes.
  6. Edit — optional, never re-runs transcription.
  7. Preview — looping playback of the reconstructed arpeggio.
Common mistake to avoid: the 3–20 second constraint belongs ONLY to Loop Selection (step 3, inside the Waveform component). The Upload component (step 1) must never show a "3–20 seconds" hint — it accepts a full audio file up to 200 MB / 10 minutes, with a warning (not a rejection) above 100 MB or 6 minutes.

Section 3 — Waveform

Displays: audio waveform, playback cursor, selection overlays.

Focus Region

Only if audio > 1 minute. Max 1 minute. Only this region is used afterwards. Skipped if audio is already short.

Loop Selection

Inside the Focus Region. Min 3s, max 20s. Draggable handles; the control itself prevents invalid lengths.

Playback controls

Play, Pause, Loop. Loops only the selected fragment.

Instruction — exact copy

Select one complete arpeggio phrase that loops seamlessly.

The better the loop,
the more accurate the analysis.

Section 4 — Analysis

One primary button, exact label: Analyze. States: Idle → Loading → Completed → Failed. During analysis: disable editing, show a spinner, status text Analyzing...

Section 5 — Results

Appear only after analysis (progressive disclosure). Reference layout, complete result:

BPM            124
Input Notes    C  D#  G
Style          Up
Rate           1/16
Octaves        2
Detected Seq.  C2 D#2 G2 C3 D#3 G3

Partial result example

BPM            ✓ 124
Rate           ✓ 1/16
Input Notes    ✓ C  ✓ E  ✓ G
Octaves        ✓ 2
Style          Not detected

Only confidently determined parameters are shown. Missing ones read Not detected — never a placeholder, dash, or guessed value.

Confidence

Exactly one of High / Medium / Low, paired with green / yellow / red. The text label must always be visible — color alone is never sufficient (accessibility requirement).

Section 6 — Manual Editing

Editable: Input Notes, Style, Rate, Octaves, BPM. Editing never re-runs transcription — it only regenerates the Detected Sequence and Preview.

Input Notes

[C] [D#] [G] [+ Add Note]

Selecting a note opens a picker: C, C#, D, D#, … plus Delete.

Style

Dropdown. Initially: Up, Down, UpDown, DownUp. Future styles load automatically from the Style Registry — no code change needed to add one.

Rate

Dropdown: 1/4, 1/8, 1/16, 1/32, 1/4T, 1/8T, 1/16T, 1/32T.

Octaves

[-]  2  [+]

BPM

124   [- ÷2]   [×2]

×2/÷2 update Rate accordingly. If halving or doubling would push Rate outside the supported list, that button is disabled (not hidden — just non-interactive).

Editing partial results

All fields stay editable on a partial result. The Preview only switches from the quantized transcription to the registry-generated sequence once all of Input Notes, Style, Octaves, BPM and Rate are set (detected or user-entered).

Section 7 — Preview

Controls: Play, Pause, Loop only (no Stop, no one-shot mode — playback always loops).

Preview and original-audio playback are mutually exclusive — starting one pauses the other, enforced by an app-level Playback Controller. A/B comparison happens by toggling between the two, not simultaneous playback.

Loading states

Upload → Ready → Analyzing → Displaying Results → Editing → Preview

Error states

Each gets its own specific message, never a generic failure:

  • Unsupported Browser
  • Unsupported Audio Format
  • Audio Decode Failed
  • No Pitched Notes Detected
  • No Repeating Arpeggio Detected
  • Arpeggiator Style Not Detected
  • Analysis Engine Unavailable
  • Unexpected Error

Responsive & accessibility

  • Desktop is the primary experience; the app is mobile-friendly with no functionality removed
  • Desktop: two-column results if space allows · Tablet: single column · Mobile: everything stacks
  • Keyboard accessible, visible focus states, semantic HTML, accessible button labels
  • Color is never the only communication channel (see Confidence above)

Animations

Allowed: fade, opacity, small transitions. Avoid: large motion, complex page transitions, decorative animation.

Notes always use sharp notation only — C# D# F# — flats (Db, Eb, Gb) are never displayed.

Non-negotiable rules

Honest UI: never invent or guess a value; unknown stays Not detected.

Single Page Application: no routing, no navigation, no modal workflows.

Progressive disclosure: Results are hidden until analysis finishes.

Minimal interface: only controls required for the current step are visible.

Every action gets immediate visual feedback; long operations always show progress.

Preview vs. original audio playback is mutually exclusive (toggle, not simultaneous).

Product boundaries (out of scope for MVP — do not design for these)

MIDI export, plugin/preset export, DAW integration, source separation, polyphonic arpeggiators, Chord Trigger, a Piano Roll editor, ADSR/filter/oscillator detection, sound recreation, AI recommendations, user accounts, cloud storage, collaboration, analytics.
